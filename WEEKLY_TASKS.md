## 九、任务 7：性能优化

### 目标

只优化真实会卡顿的场景。

### 优先处理

#### 字数统计

- [ ] 大文本输入时增加 debounce。
- [ ] 大文件解析时显示 Loading。
- [ ] 如果仍卡顿，再考虑 Web Worker。

#### YAML 工具

- [ ] 格式化/对比大文本时增加 Loading。
- [ ] 输入解析增加 debounce。
- [ ] 错误定位保持清晰。

#### 翻译工具

- [ ] 输入触发请求时增加 debounce。
- [ ] 请求中显示 Loading。
- [ ] 请求失败显示明确错误。

#### 剪贴板历史

- [ ] 历史列表过长时限制渲染数量。
- [ ] 搜索使用 debounce。
- [ ] 必要时迁移到 IndexedDB。

### 验收

- [ ] 大输入时页面不卡死。
- [ ] 用户知道当前是否正在处理。
- [ ] 优化不增加使用门槛。

## 十四、执行原则总结

- 不追求一次性重构完成。
- 不为了架构而架构。
- 不破坏静态直开能力。
- 优先解决真实体验问题。
- 每次只做一类清晰改动。
- 每个工具始终保持轻量、独立、可下载使用。
